核心数据模型

6个核心表，支撑全部业务

👤 User — 用户
id, email, name, avatar
credits: int (剩余积分)
role: user | admin
created_at, updated_at
💰 CreditTransaction — 积分流水
id, user_id (FK)
amount: +100 / -5
type: purchase | usage | gift
task_id (可选,关联生成任务)
created_at
🔥 TrendingTopic — 热门话题
id, title, description
category: challenge | music | hashtag | event
heat_score: int (热度指数)
heat_history: JSON (历史曲线)
ai_prediction: JSON (趋势预判)
platform_id (抖音话题ID)
status: rising | peak | falling
fetched_at
🎨 Content — 生成内容
id, user_id, topic_id (FK)
type: image | video
model: openai|sd|tongyi|...
prompt, negative_prompt
params: JSON (分辨率/时长等)
status: pending→processing→done|failed
file_url, thumbnail_url
metadata: JSON
created_at
📝 PublishSuggestion — 发布建议
id, content_id (FK, 1:1)
best_times: JSON (推荐时段)
caption: text (AI生成配文)
hashtags: JSON (推荐标签)
bgm: JSON (推荐BGM)
generated_at
⚙️ ModelProvider — 模型配置
id, name, type: image|video
provider: openai|sd|tongyi|wenyi|jimeng
is_active: bool
cost_per_gen: int (每次消耗积分)
config: JSON (API密钥等配置)

核心关系

User 1 ──→ N CreditTransaction    (一个用户多条积分记录)
User 1 ──→ N Content              (一个用户多次生成)
TrendingTopic 1 ──→ N Content     (一个话题可生成多条内容)
Content 1 ──→ 1 PublishSuggestion (每条内容一条发布建议)
Content N ──→ 1 ModelProvider     (多条内容使用同一个模型)
